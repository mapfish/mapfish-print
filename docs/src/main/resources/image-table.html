This section explains how tho add one or multiple images to a table subreport.
In a first step a table needs to be configured for the process !preparTable is used. To be able to resolve an image within the table the column containing the image is defined in the column element as image. The following example shows a configuration of a table where the column with the title "icon" contains the URL to the image. The column name depends on the title that is used in the interface and therefore can be the translation of the underlying database column name (if a database is used).
In the example below all columns from the original table are included in the printed table with out formating.

        - !prepareTable

          dynamic: true

          columns:

            icon: !urlImage

              urlExtractor: (.*)
               #use Regex expression to retrieve the URL of the image
              urlGroup: 1

To add formating styles to the printed table there are two possibilities:
  1. To add a rubreport in form of a jasperreport template containing all of the columns that are to be included in the printed report. This requires the dynamic flac in the process !prepareTabel to be false.
  2. Use the styling from a subreport template but generate the columns dynamically. An example for this type of configuration is given below. 
        - !prepareTable

             excludeColumns:

               - "not_shown_in_print"

             maxColumns: 8

             dynamic: true

             jasperTemplate: table_a4_portrait.jrxml   #- If dynamic is true this template will be used to obtain the column styles and the size of the subreport.

             firstDetailStyle: column_style_1          #- optional

             detailStyle: column_style_2               #- required when dynamic = true

             lastDetailStyle: column_style_3           #- optional

             firstHeaderStyle: header_style_1          #- optional

             headerStyle: header_style_2               #- required when dynamic = true

             lastHeaderStyle: header_style_3           #- optional

             columns:

               Signal: !urlImage

                 urlExtractor: (.*)

                 urlGroup: 1
